Deploy a simple static website with github pages and jekyll

Design Jekyll

How to do it (and why)

Why even bother?

Having an online presence is extremely important in the connected age. While this has become trivial thanks to social media, the capacity of deploying and managing a personal website, be it a blog or just an online resume, can make a difference in the impression that we give to other people.

Having said so, not everyone is a full stack web developer, and even if alternatives with user friendly interfaces and workflow exist (wix and square space are just a couple of examples) those tend to be bloated and expensive solutions to a simple problem.

In the majority of cases a static website can do the trick, especially when combined with the right add-ons.
This is where jekyll enters the picture.

What is jekyll?

Jekyll is a static site generator.
It takes text written in your favorite markup language and uses layouts to create a static website that is easy to tweak to your likings. In addition, there are tons of open source themes that can be picked if you want to concentrate only on the content.
For example, my blog is currently using Chalk, a beautiful and minimalistic theme made by Nielsen Ramon.

In a few steps

First things first: you’ll need a github account, from there you can create a new repository that will be your starting point for deploying on github pages (you can learn everything about it here). Once you created your repository, go in the settings tab> pages and select the current branch to deploy.

Clone the repository on your machine. You’ll need to install ruby, then bundler and jekyll.
Inside your cloned directory run $ jekyll build, this build the static site that you can push to the repository. You can also $ jekyll run the website locally to preview it.

The theme can be changed either from the settings in your repository, or modifying the _config.yml file in the directory.
While most themes are easy to install, some might not be able to be integrated with github pages out of the box.

In that case, go to the theme repository and fork it (alternatively, you can download the .zip file and extract it in your repo directory) in these cases (as it was with chalk), you’ll need to use $ npm run setup to set up the environment, $ npm run local to preview it on your machine, and $ npm run publish to create a branch compatible with github pages. In this case, the process is automatic.

That’s it, you’re done!

Now you can write your content in markdown and publish it.

I hope this very concise and quite reductive guide helped you decide if you want to invest a little time in building a static website :)

This blog is deployed on github pages using the chalk template.
